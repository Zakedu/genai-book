# A. 프롬프트 템플릿 모음

분야별 프롬프트 템플릿입니다. 상황에 맞게 수정하여 사용하세요.

---

## 학습/연구용

### 1. 논문 요약

```markdown
[역할]
당신은 학술 논문 분석 전문가입니다.

[작업]
첨부한 논문에 대해 다음을 분석해주세요:
1. 연구 질문 및 가설
2. 연구 방법론
3. 주요 발견 및 결과
4. 연구의 한계점
5. 실무/후속 연구에 대한 시사점

[형식]
- 각 항목 3-5문장으로 요약
- 비전문가도 이해할 수 있는 언어 사용
- 핵심 수치/통계는 명시
```

### 2. 개념 설명

```markdown
[주제]
{개념명}에 대해 설명해주세요.

[요구사항]
1. 한 줄 정의
2. 핵심 특징 3가지
3. 일상적인 비유를 통한 설명
4. 실제 활용 사례 2개
5. 관련 개념과의 차이점

[대상]
{대상 수준: 예) 고등학생, 대학 신입생, 비전공자}
```

### 3. 시험 대비 질문 생성

```markdown
[자료]
다음 내용을 바탕으로 시험 문제를 만들어주세요.

{학습 내용 붙여넣기}

[요구사항]
- 객관식 5문항 (각 4지선다)
- 단답형 3문항
- 서술형 2문항

[조건]
- 난이도: 중~상
- 정답과 해설 포함
- 출제 근거 명시
```

### 4. 토론 준비

```markdown
[주제]
"{토론 주제}"에 대한 토론을 준비하고 있습니다.

[요청]
1. 찬성 측 핵심 논거 3개
2. 반대 측 핵심 논거 3개
3. 각 논거에 대한 예상 반론
4. 유용한 통계/사례 자료
5. 토론 전략 제안

[추가 요청]
상대방 논거에 대한 효과적인 반박 포인트도 알려주세요.
```

### 5. 문헌 검토 도움

```markdown
[연구 주제]
{연구 주제}에 관한 문헌 검토를 진행 중입니다.

[요청]
1. 이 주제의 주요 연구 흐름 설명
2. 핵심 이론/프레임워크 소개
3. 연구 방법론별 분류
4. 연구 간극(research gap) 제안
5. 검색에 사용할 키워드 추천

[주의]
구체적인 논문명은 직접 학술 데이터베이스에서 확인하겠습니다.
```

```markdown
- {핵심 메시지 1}

{톤: 예) 정중하지만 간결하게, 진심 어린 사과}
```

### 7. 회의록 정리

```markdown
[회의 정보]
- 일시: {날짜/시간}
- 참석자: {참석자 목록}
- 목적: {회의 목적}

{회의 메모/녹취록 붙여넣기}

[요청]

1. 핵심 논의 사항 (3-5개)
```

### 8. 보고서 초안

```markdown
[보고서 유형]
{유형: 예) 주간 업무 보고, 프로젝트 현황 보고}

[보고 대상]
{대상: 예) 팀장, 임원진}

[포함 내용]
1. 핵심 성과/진행 상황
   - {내용 1}
   - {내용 2}
2. 이슈/리스크
   - {이슈 1}
3. 다음 계획
   - {계획 1}

[형식]
- 분량: A4 1페이지
- 불릿 포인트 위주
- 핵심 수치 강조
```

### 9. 기획서 구조화

```markdown
[기획 주제]
{기획 주제: 예) 신규 서비스 런칭}

[배경/목적]
{배경 설명}

[요청]
다음 항목으로 기획서 구조를 잡아주세요:
1. 목차 구성
2. 각 섹션별 포함할 핵심 내용
3. 필요한 데이터/자료 목록
4. 예상 질문과 대응 방안

[제약 조건]
- 발표 시간: {시간}
- 대상: {청중}
```

### 10. 피드백 요청

```markdown
[검토 대상]
{문서/기획안 붙여넣기}

[요청]
다음 관점에서 피드백해주세요:
1. 논리적 흐름
2. 설득력 있는 부분 / 보완 필요 부분
3. 누락된 중요 요소
4. 표현/문체 개선점
5. 전반적인 완성도 (10점 만점)

[우선순위]
가장 중요한 개선점 3개를 먼저 알려주세요.
```

## 분석/의사결정용

### 21. SWOT 분석

```markdown
[분석 대상]
{회사/제품/프로젝트}

[배경 정보]
{관련 정보 제공}

[요청]
SWOT 분석을 수행해주세요:
- Strengths (강점): 내부 긍정 요소
- Weaknesses (약점): 내부 부정 요소
- Opportunities (기회): 외부 긍정 요소
- Threats (위협): 외부 부정 요소

[추가]
각 요소별 3-5개 항목과 함께 전략적 시사점을 도출해주세요.
```

## 사용 팁

### 템플릿 커스터마이징

1. **{중괄호}** 부분을 실제 내용으로 대체
2. 불필요한 항목은 삭제
3. 필요한 항목은 추가
4. 구체적일수록 좋은 결과

### 효과적인 사용법

| 팁 | 설명 |
|----|------|
| 맥락 제공 | 배경 정보를 충분히 제공 |
| 구체적 지시 | 모호한 표현 피하기 |
| 형식 지정 | 원하는 출력 형식 명시 |
| 반복 개선 | 결과물 기반으로 프롬프트 수정 |
